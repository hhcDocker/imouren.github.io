## sentry 安装

文档地址：https://docs.sentry.io/server/installation/python/

依赖安装

```python
yum install -y python-setuptools python-pip  libffi-devel libjpeg-devel libxml2-devel libxslt-devel libyaml-devel python-devel postgresql-devel postgresql-libs

redis 要求 2.8.9以上
```

虚拟环境安装

```python
# 安装包
pip install virtualenv

pip install virtualenvwrapper

# 查看路径
which virtualenvwrapper.sh

# 添加环境变量
vim ~/.bash_profile

append:

export WORKON_HOME=/opt/workon_home/
source /usr/local/bin/virtualenvwrapper.sh

# 目录权限修改
chown -R admin:admin workon_home/

# 生成虚拟环境
mkvirtualenv sentry -p python2.7

# 安装sentry
pip install sentry==8.8 -i http://pypi.douban.com/simple --trusted-host pypi.douban.com

运行sentry试下，验证以及安装成功

# 其他需要的python包
supervisor
mysql-python
```

mysql配置

```python

CREATE DATABASE `sentry` character set utf8;
create user 'sentry'@'localhost' identified by 'sentry';
grant all on `sentry`.* to sentry@'localhost';
FLUSH PRIVILEGES;
```

初始化配置文件

默认在 `~/.sentry`下，可以指定位置生成`sentry init /etc/sentry`

会有两个文件：config.yml  sentry.conf.py

配置mysql和redis

```python

# /etc/sentrysentry.conf.py

DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.mysql',
        'NAME': 'sentry',
        'USER': 'root',
        'PASSWORD': 'root',
        'HOST': '192.168.28.209',
        'PORT': '',
    }
}

# /etc/config.yml
redis.clusters:
  default:
    hosts:
      0:
        host: 192.168.28.208
        port: 6379
        # password: "my-secret-password"
```

配置邮箱

```python

mail.from: 'sentry@localhost'
mail.host: 'localhost'
mail.port: 25
mail.username: ''
mail.password: ''
mail.use-tls: false

或者不使用邮箱

mail.backend: 'dummy'
```

如果需要启用SSL

```python
# /etc/sentrysentry.conf.py

SECURE_PROXY_SSL_HEADER = ('HTTP_X_FORWARDED_PROTO', 'https')
SESSION_COOKIE_SECURE = True
CSRF_COOKIE_SECURE = True
```

运行migrations

```python
SENTRY_CONF=/etc/sentry sentry upgrade

SENTRY_CONF=/etc/sentry sentry upgrade django migrate sentry --fake 0229 # 对mysql支持不好，上面会报错，手工执行
```

创建超级用户

```python
SENTRY_CONF=/etc/sentry sentry createuser
```

单独启动服务

```python

# web
SENTRY_CONF=/etc/sentry sentry run web  # http://localhost:9000/

# works
export C_FORCE_ROOT="true"  # root可以用了

SENTRY_CONF=/etc/sentry sentry run worker

# cron process
SENTRY_CONF=/etc/sentry sentry run cron
```

简单配置nginx

```python

location / {
  proxy_pass         http://localhost:9000;
  proxy_redirect     off;

  proxy_set_header   Host              $host;
  proxy_set_header   X-Forwarded-For   $proxy_add_x_forwarded_for;
  proxy_set_header   X-Forwarded-Proto $scheme;
}
```

使用uwsgi
```python

# sentry.conf.py
SENTRY_WEB_OPTIONS = {
    'protocol': 'uwsgi',
}

# nginx

server {

  location / {
    include     uwsgi_params;
    uwsgi_pass  127.0.0.1:9000;
  }
}

```

配置 supervisord
```python
# /etc/supervisord.conf

[unix_http_server]
file = /var/run/supervisord.sock
chmod = 0777
chown= root:root

[supervisorctl]
serverurl = unix:///var/run/supervisord.sock

[supervisord]
pidfile=/var/run/supervisor.pid
logfile=/var/log/supervisor.log
logfile_maxbytes=128MB
loglevel=ERROR
user=root

[inet_http_server]
port=*:9001
username = admin
password = admin

[rpcinterface:supervisor]
supervisor.rpcinterface_factory = supervisor.rpcinterface:make_main_rpcinterface

[program:sentry-web]
directory=/opt/workon_home/sentry/
environment=SENTRY_CONF="/etc/sentry"
command=/opt/workon_home/sentry/bin/sentry start
autostart=true
autorestart=true
redirect_stderr=true
stdout_logfile=syslog
stderr_logfile=syslog

[program:sentry-worker]
directory=/opt/workon_home/sentry/
environment=SENTRY_CONF="/etc/sentry"
command=/opt/workon_home/sentry/bin/sentry run worker
autostart=true
autorestart=true
redirect_stderr=true
stdout_logfile=syslog
stderr_logfile=syslog

[program:sentry-cron]
directory=/opt/workon_home/sentry/
environment=SENTRY_CONF="/etc/sentry"
command=/opt/workon_home/sentry/bin/sentry run cron
autostart=true
autorestart=true
redirect_stderr=true
stdout_logfile=syslog
stderr_logfile=syslog


```

启动

```python
supervisord -c /etc/sentry/supervisord.conf

supervisorctl start|stop all

supervisorctl reload

supervisorctl stop|start sentry-web

```

```python
$ crontab -e

```
